# Tutorial para crear una tienda online en Wordpress

Repositorio documental para replicar una tienda online real en WordPress usando Pantheon, WooCommerce, Astra y Elementor, con apoyo de IA.

## 🎯 Objetivo

Este repositorio se ha creado para reproducir esta tienda online de referencia:

- https://dev-noe-dam.pantheonsite.io/

La idea no es solo leer documentación, sino usar este material para llegar a un resultado concreto y verificable: construir una réplica funcional de esa tienda.

Para hacerlo, el repositorio reúne todo lo necesario para que un alumno pueda aprender y ejecutar el proceso apoyándose en:
- documentación curada
- contenido textual estructurado en `tutorials/`, `how-to/`, `reference/` y `explanation/`
- activos visuales
- una IA que use estos materiales como fuente principal de verdad

Permite:
- reducir el ruido de la IA  
- ajustar los contenidos al nivel del alumnado  
- trabajar con guías basadas en el uso real de la herramienta
- aprender siguiendo un resultado observable y reproducible

## 📚 Estructura

### Estructura actual

- `tutorials/` → recorridos guiados de aprendizaje  
- `how-to/` → tareas concretas  
- `reference/` → pantallas, campos y opciones  
- `explanation/` → conceptos y contexto  
- `assets/` → imágenes, iconos, etc

### Activos disponibles

Dentro de `assets/` hay materiales que complementan la documentación:

- `assets/images/` → imágenes de productos, iconos y recursos visuales

Estos activos no son solo material de apoyo: forman parte del contenido necesario para replicar la tienda de referencia y pueden cargarse junto con los Markdown en herramientas como NotebookLM.

## 🧩 Qué se construye con este repositorio

La propuesta de este repositorio es:

- `aprende a crear esta tienda`

No se trata solo de leer documentación, sino de ser capaz de reproducir un resultado.

En este caso, el resultado esperado es una tienda creada con:
- WordPress

Para validar esto, puede usarse:

Además, el resultado final puede incorporarse al porfolio del estudiante como una muestra práctica de trabajo con WordPress, WooCommerce, Astra, Elementor y Pantheon, lo que puede reforzar su empleabilidad de cara a empresas que busquen perfiles capaces de construir y adaptar tiendas online reales.

## 🤖 Uso con IA

### Uso recomendado con NotebookLM

Este repositorio está pensado para ser usado directamente por una IA, especialmente dentro de un cuaderno de NotebookLM.

La forma de uso recomendada es:

1. Crea un cuaderno en NotebookLM para este repositorio.  
2. Sube los archivos Markdown del repositorio.  
3. Sube también los activos relevantes de `assets/`, especialmente imágenes.
4. Indica a la IA que use estos materiales como fuente principal de verdad.  
5. Empieza con esta pregunta: `¿Qué voy a aprender con este cuaderno?`
6. Después, indícale de forma explícita que este tutorial sirve para crear una réplica idéntica de esta tienda de referencia: `https://dev-noe-dam.pantheonsite.io/`.
7. Pídele que, cuando explique el objetivo del cuaderno o del tutorial, devuelva también ese enlace a la tienda de referencia.
8. Pídele que guíe al alumno usando primero `tutorials/`, después `how-to/`, `reference/` y `explanation/` según haga falta.  
9. Si algo no aparece en las fuentes, pídele que lo diga claramente.  
10. Haz preguntas concretas y orientadas a tarea.  

### Orden sugerido para la IA

Si el alumno no sabe por dónde empezar, la IA debería apoyarse primero en:

- `tutorials/crear-una-tienda-basica-en-wordpress-y-woocommerce.md`
- `tutorials/crear-un-catalogo-basico-en-woocommerce.md`

Además, la IA debería dejar claro desde el principio que el objetivo del repositorio es reproducir esta tienda:

- https://dev-noe-dam.pantheonsite.io/

Y después usar:

- `how-to/` para ejecutar tareas concretas
- `reference/` para resolver dudas sobre pantallas, botones y opciones
- `explanation/` para aclarar conceptos

## 📌 Política de uso con IA

Este repositorio incluye una política específica para herramientas de IA en:

- `AI-USAGE-POLICY.md`

La idea es que la IA:
- use estos Markdown como fuente principal
- indique claramente cuando una respuesta no procede de estos archivos
- diferencie la información del repositorio de la información obtenida en fuentes externas

## 🛠️ Mantenimiento documental

Este repositorio incluye una guía de mantenimiento en:

- `DOC-MAINTENANCE.md`

Su función es fijar:
- qué carpeta debe contener cada tipo de información
- qué archivos son fuente canónica
- qué repeticiones se deben evitar

## 🧠 Enfoque

- Separación entre conceptos y práctica  
- Documentación curada  
- Uso guiado de IA en el aprendizaje  
- Aprendizaje orientado a un resultado final reproducible

Una vez aprendida la réplica base, se recomienda sofisticar y personalizar la tienda para convertirla en un proyecto más propio. Se anima a modificar contenidos, ajustar la identidad visual, introducir mejoras y usar imágenes distintas para que el resultado final no sea solo una copia técnica, sino también una pieza más valiosa para el porfolio.

También se anima a probar variantes tecnológicas a partir de esa base: usar otros temas distintos de Astra, como Divi, trabajar con maquetadores diferentes de Elementor, añadir nuevos plugins e incluso desplegar la tienda en otros servicios de hospedaje distintos de Pantheon. La idea es que, después de dominar esta réplica, el estudiante pueda extrapolar lo aprendido a contextos y decisiones técnicas diferentes.

## 🧭 Organización del contenido

La documentación se ha estructurado siguiendo Diátaxis:

- `explanation/` responde sobre todo a `qué es`, `por qué` y `cómo se relacionan los conceptos`
- `how-to/` responde a `cómo hacer una tarea concreta`
- `reference/` responde a `qué opciones, pantallas o campos existen`
- `tutorials/` responde a `cómo aprender haciendo`

La estructura anterior ya ha sido absorbida en esta organización nueva para evitar duplicidades y simplificar el mantenimiento.

## 🏷️ Convención de nombres

Los nombres de archivo siguen estas pautas generales:

- `how-to/` usa verbos en infinitivo orientados a tarea
- `reference/` usa nombres de pantalla, bloque o listado consultable
- `explanation/` usa nombres de concepto, comparación o contexto
- cuando aporta claridad, se añade el contexto en el nombre, por ejemplo `-en-wordpress` o `-en-ecommerce`
